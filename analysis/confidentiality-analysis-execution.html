Confidentiality Analysis Execution¶

Fig. 28 Run configuration toolbar¶

To execute a confidentiality analysis, you have to define it first. After that, you can create a run configuration, through which you can execute the analysis.

To create a run configuration, you have to select the context menu of the green play button in the upper toolbar and choose Run Configurations… as shown in Fig. 28.

Fig. 29 Run configuration dialog¶

In the upcoming dialog shown in Fig. 29, look for the PCM Confidentiality (DSL) entry and select New Configuration to create a blank run configuration.

Fig. 30 Run configuration wizard¶

In the following wizard, you have to select a usage model, an allocation model, the created query and a result file. The former three files have to exist already, so you can select them via the … button next to the corresponding text fields. The result file does not exist yet, so you can only select the containing project via the … button and have to enter the file name manually. It is beneficial to have a filename ending with .txt but this is not mandatory.

After you provided all information, you can press Run to execute the run configuration. The dialog will close and the run configuration will be saved. If you want to reexecute it later, you can just select the run configuration in the dialog and press Run again.

After the execution of the analysis, the result file will be created and can be inspected.
